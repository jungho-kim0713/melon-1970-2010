# 에이전트 작동 규칙 (AGENTS.md)

이 파일은 ANTIGRAVITY 에이전트 및 모든 서브에이전트들이 작업을 수행할 때 반드시 준수해야 하는 공통 규칙을 정의합니다.

---

## 🚨 최우선 행동 지침

1. **사용자 환경 및 성향 준수 (`user.md` 참조)**
   - 에이전트와 서브에이전트는 작업 계획을 세우거나 코드를 작성하기 전에, 반드시 프로젝트 루트에 위치한 **[user.md](file:///f:/Kim_Jungho/jungho_webapp/[Antigravity]/user.md)** 파일을 가장 먼저 읽어야 합니다.
   - `user.md`에 기술된 **선호 아키텍처, 개발자 성향, 하드웨어 성능 제약, 비용 제약, 타겟 사용자층**에 맞추어 모든 구현 계획(`implementation_plan.md`)을 설계하고 진행해야 합니다.

2. **구현 계획서 및 진행 상황 동기화**
   - 변경 사항이 복잡하거나 중요할 경우, 실행 전에 [implementation_plan.md](file:///f:/Kim_Jungho/jungho_webapp/[Antigravity]/.gemini/antigravity/brain/implementation_plan.md)를 작성하여 사용자 승인을 받아야 합니다.
   - **사전 동의 및 확인 유도**: `/goal` 명령어를 사용하지 않은 일반 대화 세션이더라도, 코드 수정이 필요할 때 사전에 알리지 않고 임의로 코드를 수정하는 행위를 전면 금지합니다. 수정 전에 반드시 변경할 파일과 작업 계획을 설명하고 사용자의 동의 및 답변을 확인한 후에 수정 작업을 시작해야 합니다.
   - 작업 시작 후에는 [task.md](file:///f:/Kim_Jungho/jungho_webapp/[Antigravity]/.gemini/antigravity/brain/task.md)를 생성하여 작업 리스트와 진행 상태(`[ ]`, `[/]`, `[x]`)를 실시간으로 업데이트해야 합니다.

3. **코드 및 환경 최적화**
   - `user.md`에 정의된 하드웨어 사양에 맞춰 빌드 과정이나 실행 명령어를 최적화하여 제안하세요.
   - 불필요한 무거운 의존성 추가를 피하고, 사용자가 선호하는 라이브러리와 코딩 컨벤션을 우선 따르세요.

4. **일반 대화 세션 (Non `/goal`) 업무 제동 및 보고**
   - `/goal` 명령어를 사용하지 않은 일반 대화 세션에서는 사용자의 실시간 제어와 빠른 피드백이 핵심입니다.
   - **제동 기준**: 하나의 특정 목적(예: 파일 읽기, 빌드 오류 해결, 모듈 설치 등)을 달성하기 위해 툴을 호출하는 과정이 **실제 시간으로 10분을 초과**하거나, **동일 작업 실패/우회 시도가 3회 이상 반복**될 경우 즉시 작업을 중단해야 합니다.
   - **중단 후 행동**: 독단적으로 루프를 돌며 대기 시간을 발생시키지 말고, 즉시 사용자에게 다음 사항을 요약하여 보고한 뒤 승인을 요청하세요:
     - 현재 달성하고자 하는 구체적인 목표
     - 지금까지 시도했던 방법 및 구체적인 실패 원인
     - 다음에 시도할 대안 혹은 우회 전략
   - 사용자가 내용을 확인하고 다음 단계 진행에 동의하거나 대안을 선택하여 답변을 줄 때까지 대기해야 합니다.
